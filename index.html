A lightning fast framework for mocking Spring Data repositories

Spring Data provides a great foundation for separating data from business concerns. However, when you want to test your application, it forces you to either write lots of boilerplate code or start a full blown application context and backing database. This wastes expensive developer time, makes tests run slower, and is completely unnecessary.

Spring Data Mock will save you time and money, while making your tests run faster.

Reduce the amount of code you need to write, while using a reliable infrastructure that replicates what Spring would do with an actual database, only in-memory. Spring Data Mock is open source and available for download from GitHub and Maven Central. Take a look at our docs for instructions on getting started.

Team

Hey, I'm Milad Naseri. I'm a professional software engineer and TDD enthusiast from Kirkland, Washington USA.

A few years ago, I was testing some pretty complicated services that relied on Spring Data. It was a huge hassle to keep the test environment up-to-date with the requirements and real world situation. After a long struggle, I wrote Spring Data Mock so that I could isolate my services and test them regardless of database features.
